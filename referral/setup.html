Referral portal setup

Sign-in, sign-up, and dashboard in a few steps

⚡ Simpler option: internal auth (no Supabase)

Skip Supabase entirely. Add these Netlify env vars: REFERRAL_USE_SIMPLE_AUTH=true, REFERRAL_AGENTS (username:password:CODE per agent), REFERRAL_SECRET. See REFERRAL_SIMPLE_AUTH_SETUP.md in the repo.

1Create a Supabase project

If you don’t have one yet:

Supabase → New project

Pick a name, set a database password, choose a region → Create. Wait until the project is ready.

2Run the database schema

Copy the SQL below, then in Supabase open SQL Editor → New query → paste → Run.

You should see “Success. No rows returned.” This creates tables and the signup trigger.

3Get your API keys

In Supabase: Settings (gear) → API.

Copy and keep ready: Project URL and the anon public key.

4Set Auth redirects

In Supabase: Authentication → URL Configuration.

  • Site URL: your live site, e.g. https://jlsolutions.io
  • Redirect URLs: add https://jlsolutions.io/** and http://localhost:8888/** if you test locally
5Add Netlify environment variables

In Netlify: your site → Site configuration → Environment variables.

Add two variables (all scopes):

  • SUPABASE_URL = your Project URL
  • SUPABASE_ANON_KEY = your anon public key
6Redeploy

Netlify → Deploys → Trigger deploy → Deploy site. The referral config is read from env on each request after deploy.

7Test

Sign up, then sign in:

Sign up Sign in Dashboard

If sign-up asks to confirm email, either confirm it or turn off “Confirm email” under Authentication → Providers → Email (for testing).
